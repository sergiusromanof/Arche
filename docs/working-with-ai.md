# Working with the assistant

A short guide to getting the most out of an AI assistant on real tasks.
This is guidance for you, the person — it is not installed as a rule.

## Before you start

Describe the task first — in plain words, pseudo-code, and diagrams or graphs where they help.
Putting the shape of the solution into words and pictures surfaces the gaps and the decisions before any code is written.

## While planning

Talk the task through with the assistant and understand its proposed implementation step by step, in detail, before building.
Then step back and prune: what can be removed, do you actually need all of it, and is there a simpler or more efficient way?

## Sequence the work from easy to hard

Start with the easy parts and move step by step toward the hard ones.
Split a hard part into simpler pieces, order them from simplest to hardest, and work through them the same way.
Make each step its own commit.

## Learn from the work

Problems repeat, so remember what you learn.
Notice which problems keep coming up and on which kinds of tasks, and capture the fix — as a rule, a command, a config value, or a memory — so the next run starts ahead of the problem.
