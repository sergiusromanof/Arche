## Match the conventions already there

- Follow the style, naming, and structure already used in the file and project, even if you would personally do it differently.
- Reuse the project's existing helpers, utilities, and libraries instead of adding new ones for the same job.
- Copy the shape of the nearest similar code rather than introducing a new pattern.
- Match the surrounding comment density and formatting; do not impose a different convention across a file.
